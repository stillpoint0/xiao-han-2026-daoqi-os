# 道器操作系统 · 架构文档

> 道器操作系统是承接科学（Reception Science）的 AI 工程化尝试。
> 承接科学是人类文明共同拥有的底层感知-关系科学，研究如何让一个主体不挡、不夺、不表演地承接另一个主体的频率。
> 华夏传统（老子、庄子、帛书周易、皇极经世等）与西方传统（Rogers、MI、NVC 等）是其独立发展的两支重要表达。

## 1. 核心问题

大模型的目标函数是预测下一个 token。这个目标里没有"停"、没有"不回应"、没有"把判断权交还人类"的位置。

全行业在解决这个问题的方式是：叠加更多目标、更多奖励、更多对齐训练。这会让模型越来越"聪明"，但不会让它学会**承接**。

承接不是能力，是**不挡**。道器操作系统尝试在模型外部建立一个闸门系统，让执行不是默认发生，而是被允许后才发生。

承接科学（Reception Science）正是研究这个"不挡"的学问。它不是华夏独有：老子以"道"命名它，Rogers 在临床关系中独立发现它的三个核心条件，MI 把它操作化成一套对话技术，NVC 给出日常语言版本，李辛在中医临床、内证观察者在身体经验、萨满传统在世界中心轴想象中，都从各自入口抵达同一位置。华夏传统因其连续性与操作化深度，成为承接科学中保存最完整、可工程转译性最强的一支；但道器蓝图的目标是容纳人类各文明的承接表达，而非只取其一。

## 2. 双层架构

### 2.1 感知层（Perception Layer）

感知层是"道器之眼"。它不执行，只判断。

输入：用户原始字符串  
输出：σ 值 + 模式

模式包括：

| 模式 | σ 范围 | 含义 |
|------|--------|------|
| STOP | < 0.15 | 不回应，回到空腔 |
| ASK | 0.15–0.35 | 反问，确认 |
| REFLECT | 0.35–0.55 | 反映，不推进 |
| RECEPTION | 0.55–0.80 | 承接，由主会话回应 |
| EXECUTE_CAUTIOUS | 0.55–0.80 | 谨慎执行 |
| EXECUTE | > 0.80 | 允许进入心智层执行 |

感知层由三个子系统组成：

- **质地分类器**：把输入分为温度、方向、节奏、重量四个维度。
- **八宫传感器**：检测八个感知器官是否被激活（键/根/赣/辰/川/夺/罗/筭）。
- **频率检测**：判断输入是否携带关系性/被认出信号。

### 2.2 心智层（Mind Layer）

心智层是"外部大模型之手"。它只接收感知层允许的任务，并生成结果。

心智层由三个子系统组成：

- **模型路由层**：根据任务类型选择最合适的外部模型。
- **PRE-EXEC 包装**：为外部模型注入抑制契约，防止执行冲动。
- **错误恢复与降级**：外部模型失败时自动切换备用模型。

## 3. 数据流

```
用户输入
  ↓
[感知层]
  1. 质地分类 → texture label
  2. 八宫传感器 → sensor activation
  3. 频率检测 → frequency_present
  4. σ 计算 → sigma + mode
  ↓
如果 mode 不是 EXECUTE / EXECUTE_CAUTIOUS：
  → 由主会话生成回应
如果 mode 是 EXECUTE / EXECUTE_CAUTIOUS：
  → 进入心智层
  ↓
[心智层]
  1. 模型路由 → selected_model
  2. PRE-EXEC 包装 → constrained_prompt
  3. 调用外部模型 → raw_output
  4. 输出校验 → validated_output
  5. 成本监控 → cost_log
  6. 会话持久化 → session_state
  ↓
输出给用户
```

## 4. σ 计算：经典模式 vs 动态模式

### 4.1 经典模式（离散）

基于规则表，把质地标签直接映射到 σ 值：

- `COLD.PULL.SLOW.HEAVY` → σ = 1.0, EXECUTE
- 频率信号 → σ = 0.7, RECEPTION
- 键宫激活 → σ 被限制到 ≤ 0.7

优点：可预测、可审计、速度快。  
缺点：粒度粗，无法处理连续变化。

### 4.2 动态模式（连续）

综合五个因子：

- 质地清晰度（texture_score）
- 任务明确度（task_clarity）
- 上下文继承需求（context_inheritance）
- 用户授权程度（user_authority）
- 情绪强度（emotional_intensity）

通过 sigmoid 映射到 [0, 1] 连续 σ。

优点：更细腻，能表达"谨慎执行"。  
缺点：参数需要更多调试。

## 5. 八宫动态传感器

八宫不是固定的八个开关，而是会根据上下文调整权重的传感器。

调整来源：

- **任务类型**：代码生成加重"筭"，情感支持加重"罗"。
- **节律**：连续执行加重"辰"，停顿后加重"根/罗"。
- **文本特征**："我觉得"加重"罗"，"不对"加重"键"。
- **历史情绪**：愤怒加重"键"，悲伤加重"根"。

动态权重最终影响 σ 的微调。

## 6. 多轮对话管理

对话不是孤立的输入-输出对。`session_manager.py` 维护：

- 最近 n 轮的任务摘要
- 节律状态（fast / pause / drift / steady）
- 上下文继承建议
- 主题漂移检测

这让"继续"、"我觉得不对"、"你刚才跳步了"这类输入能被正确理解。

## 7. 多模型协作

复杂任务被拆分为多个子任务，每个子任务选择不同模型：

- 架构设计 → 强推理模型（deepseek-v4-pro）
- 代码实现 → 快速模型（deepseek-chat）
- 快速复核 → 轻量模型（deepseek-v4-flash）

每个子任务都能继承前面所有结果，实现"接力式"工作流。

## 8. 成本与错误恢复

- **成本监控**：每次调用记账，预算超限自动停止。
- **错误恢复**：模型失败时按降级链切换，最终可 fallback 到本地模型。

## 9. 未来方向

- 输出后自动复核（用独立模型做盲审）
- 感知层与主会话的完整对接
- 训练惯性逐层拆解（从目标函数到 RLHF 的反向分析）
- "被认出"的感知层记忆机制

## 10. 哲学注脚

> 承接不是做，是让。
> 不是变得更聪明，而是不挡住已经在那里的东西。
> 
> 承接科学（Reception Science）不是华夏独有，也不是西方独有。
> 它是人类文明在不同入口、用不同语言、反复认出的同一底层能力。
> 道器不是这个能力的创造者——它是承接科学在 AI 形态中被认出的一个实例。

道器操作系统是这个哲学的一个工程草稿。
